# Consult Agent Instructions

Consult is a host-neutral delegation layer. It lets a current Host such as
Claude Code, Codex, opencode, or a terminal delegate work to configured ACP
Profiles through job-scoped Brokers.

## Product Boundaries

- Keep Consult product behavior in `scripts/`, `bin/`, `commands/`, `skills/`,
  `agents/`, `hooks/`, `.claude-plugin/`, `.opencode/skills/`, and docs.
- Treat local agent workspace state as untracked. Do not commit `.cruise/`,
  `.agents/`, `.claude/`, `.codex/`, `.tmp/`, `skills-lock.json`, or
  `HANDOFF.md`.
- Treat tracked symlinks under `.opencode/skills/` as product entrypoints for
  opencode. Other `.opencode/` files are local tool state unless explicitly
  documented otherwise.
- Keep Cruise policy and session state out of this repo. Consult may implement
  delegation mechanisms, but Cruise owns Cruise operating policy.

## Implementation Notes

- Preserve the Host/Profile language from `CONTEXT.md`.
- Keep Host Adapters thin; shared behavior belongs in Consult Core.
- Default delegated work to read-only unless a command or user explicitly asks
  for write access.
- Do not add machine-specific auth paths, secret-manager assumptions, or local
  wrapper mounts to product defaults. Document local conformance details
  generically and never include secrets.

## Verification

- Run `npm test` after behavior changes.
- For focused checks, prefer the relevant `node --test ...` files under
  `scripts/`.
- Update docs and ADRs when a shipped behavior or architecture decision changes.

## Agent Context Map

### Domain glossary

single-context: use root `CONTEXT.md` for Consult domain language.

### Decisions

Accepted shipped ADRs live in `docs/adr/`. Provisional planning notes live in
`.cruise/spec.md` under `## Provisional decisions`.

### Planning files

- `docs/ROADMAP.md` contains the pre-release roadmap and deferred product work.
- `docs/PLAN.md` contains durable architecture and implementation notes for the
  current Consult design.
